Introduction

Project

Our project’s title is ‘Reece 3D Planner’. This objective of this project is to build an application which allows users to create and design their own bathrooms. The program aims to allow users to enter the pre-defined bathrooms using first-person controller or third-person controller and select, drag, rotate the bathroom furnitrues on their own. In addtion, user could change the color of each furniture through a color palette. Up to now, the ‘Reece 3D Planner’ is accomplished and the functionality of this application meets the client’s expectation.

Role

My role is to design and implement the UI elements for this application, including main menu and in-game menus. Therefore, it is my job to guarantee the layout of each menu meets client’s needs, and all scenes should be switched properly and rapidly. Another responsibility for me is to do code optimization to make my UI system extendible and compatible, qualified with fantastic user experience.

Assertion

In general, my individual contribution to this project consists of two main parts:

  • Develop the UI system
  • Lead the team to accomplish the planner in the second half of capstone

UI system

In the beginning, I was allocated to complete the UI components. Therefore, it is my job to guarantee all UI elements available and meet the client’s requirements. Specifically, I designed the main menu and in-game menus for this project.
For the main menu, initially I did not consider the layout and color settings. The only purpose was to set up a menu enabling users to switch scenes smoothly.

As time went by, based on the client’s feedback, I started to think about the overall aesthetic aspect seriously. After several iterations, I changed the background and added new materials & animations for each panel. The last version of the main menu looks more eye-catching and clearer.

The in-game menus are the same. To enhance user experience, I replaced plenty of buttons in my later prototype, and moved several panels’ locations in order to make the context simpler and easier to recognize.

Leadership

In the second half of capstone, I took the responsibility as a leader. To organize the team, I reassigned each member’s role, making sure that all members could finish their tasks to the best advantage. Then, to guarantee that the project stays on schedule, I set the deadline of each sub task to teammates on Trello.

Moreover, because I am more familiar with the Unity, I helped the team develop each room’s light system as well. In Sprint 4, one of our members had a terrible accident and she was unable to continue contributing to our project. In consequence, I substituted her and designed the light components so as to make sure our project would not be delayed.

During this period, most members worked industriously and now our project is finished as expected. In the end, I received good feedback from them.

Summary

In conclusion, as a UI designer and a group leader in the second half of capstone, I contributed to our project through designing & implementing its UI system independently and organizing the group to complete the project efficiently. With my help, our planner is now able to be deployed on time, with fully operational and functional UI elements.
